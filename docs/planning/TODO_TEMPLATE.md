# Project TODO

> 只记录当前开放、能独立验收的高价值工作。完成历史由版本控制保存，不在本文件累积。

## 收录标准

任务必须至少满足一项：

- 提供一项完整的用户或开发者能力。
- 改变跨模块契约、生命周期或责任边界。
- 阻断发布，或涉及安全、数据完整性和故障恢复。

局部样式、文件改名、普通重构、单个测试和已经完成的工作不进入 TODO。

## P0 · 当前阻断

没有阻断时写一句“当前没有已确认的 P0 阻断”，不要创建占位复选框。

<!-- 示例：- [ ] `AREA-001` 描述阻断、完成结果和验收边界。 -->

## P1 · 当前主线

- [ ] `AREA-001` 描述本阶段必须交付的完整能力，以及可以验证完成的结果。

## P2 · 后续验收

- [ ] `AREA-002` 描述主线完成后需要闭环的生产验收或分发能力。

## 维护规则

- 开放任务总数不超过 12 项。
- 一项任务代表一个完整结果，不把每个实现步骤拆成独立任务。
- 描述必须包含验收边界，避免只写“优化”“完善”或“继续处理”。
- 任务完成后直接删除，不保留“已完成”章节。
